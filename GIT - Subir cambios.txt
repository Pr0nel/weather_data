2do En git bash

git pull :Trae la ultima actualizacion del repo en GITHUB que han hecho los otros desarrolladores.

git add [--all | -A] :Sube a un area previa (staging area) todas las modificaciones en el repository local.
	
	git add . :Sube los nuevos archivos y modificaciones, sin eliminaciones (en el directorio actual y sus subdirectorios).
	
	git add [-u | --update] :Sube las modificaciones y eliminaciones, sin nuevos archivos.

git commit -m "descripcion" :Confirmacion del cambio que se va a subir (Se va a pasar del staging area al repository) y se pone una etiqueta "descripcion".
	
	git config --global user.email "you@example.com" :Asocia el correo del usuario que ha hecho los cambios en el código
	
	git config --global user.name "Your name" :Asocia el nombre del usuario

git push origin main :Manda a la nube (en GITHUB) en la rama llamada main. (Se usa para subir a un repository remoto)

Con esto ya se subio correctamente los cambios en GITHUB

3ro Verificar los cambios en la nube o local

git status :Estado del repo
Existen 3 fases: 
Rojo (borrador) recien se esta editando
Verde (previo a confirmar cambios) ya se hizo git add
Sin color (ya se sincronizo) local y nube tienen la misma informacion

git init :Le vamos a decir al SO que vamos a iniciar un proyecto nuevo en git.

git clone :Hace una copia desde el servidor central donde está el código a tu computador.

git log :Para ver los commit que hemos creado.
